## Claude Code 进度感知方案

### 问题分析
当前 `_run_claude_code` 使用 `process.communicate()` 等待进程结束，无法实时获取输出。用户对 Claude Code 工作完全无感知。

### 解决方案
利用现有的 SSE 流式推送机制，实时推送 Claude Code 的 stdout 输出。

### 修改文件

**1. `nanobot/claude_code/manager.py`**
- 添加 `progress_callback` 参数到 `start_task()` 和 `_run_claude_code()`
- 改用流式读取 stdout（`process.stdout.readline()`）
- 实时调用回调推送进度事件

**2. `nanobot/agent/tools/claude_code.py`**
- 添加 `progress_callback` 参数
- 传递给 manager

**3. `nanobot/agent/loop.py`**
- 在 `_process_message` 中传递 `progress_callback` 给 ClaudeCodeTool

### 进度事件格式
```json
{
  "type": "claude_code_progress",
  "task_id": "bc5b0335",
  "line": "Creating file: css/style.css",
  "timestamp": "2026-02-14T15:34:35"
}
```

### 前端显示
利用现有的 `ToolStepsPanel` 组件，在 tool 运行时显示 Claude Code 的输出行。

### 预估代码量
- manager.py: ~50 行修改
- claude_code.py: ~10 行修改
- loop.py: ~5 行修改
